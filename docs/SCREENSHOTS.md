# SableOS Screenshot Guide

Screenshots help show what SableOS looks like, how modules behave, and how the command environment is organized.

Because SableOS is built from a real personal workstation, screenshots must be reviewed carefully before being added to the repository.

## Purpose

The `screenshots/` folder is for safe, intentional images used in documentation, README files, walkthroughs, and future project presentation.

Screenshots should help explain the system without exposing private information.

## Good Screenshot Examples

Useful screenshots may include:

- Clean desktop overview
- Terminal running a SableOS module
- Ship Doctor output
- Control Center interface
- Widget layout examples
- Theme previews
- Multi-monitor layout diagrams
- Before/after desktop customization examples

## What to Avoid

Do not add screenshots that show:

- API keys
- tokens
- passwords
- private emails
- calendar events
- student information
- school documents
- personal messages
- private file paths
- browser tabs with personal accounts
- local network addresses
- SSH keys or terminal history with secrets
- unfinished messy folders unless intentionally documented

## Screenshot Review Checklist

Before committing a screenshot, check:

- [ ] No private messages are visible
- [ ] No student or school-sensitive information is visible
- [ ] No personal email or calendar data is visible
- [ ] No API keys, tokens, passwords, or secrets are visible
- [ ] No unnecessary browser tabs are visible
- [ ] No private folders or file names are exposed
- [ ] The image supports the documentation
- [ ] The image is not too large for the repository
