# SableOS v0.3.0

Preview-only Ship Mode release.

## Summary

SableOS v0.3.0 introduces the first preview-only Ship Mode command path through the public `sableos` command hub.

This release adds safe mode planning commands without changing the live desktop.

## What Changed

SableOS now includes preview-only Ship Mode support through:

```bash
sableos mode list
sableos mode current
sableos mode preview work
sableos mode preview lounge
sableos mode preview clean
sableos mode preview engineering
```

These commands describe planned desktop modes, but they do not apply them yet.

## Ship Modes

### Work

Focused productivity, planning, communication, writing, task management, and deep work.

### Lounge

Relaxed desktop use, media, browsing, ambient use, personal time, and low-pressure creative work.

### Clean

A minimal reset mode designed to reduce visual clutter and return the desktop to a quiet baseline.

### Engineering

Technical work, development, diagnostics, system tools, local services, logs, and build workflows.
## Safety

This release is preview-only.

Ship Mode does not change:

* Wallpapers
* Widgets
* Startup apps
* Cinnamon settings
* Panels
* Windows
* Displays
* Running processes

The goal is to establish the command structure safely before allowing real desktop changes in a future release.

## Added

* Added `modules/ship-mode/bin/ship-mode`
* Added preview-only Ship Mode command behavior
* Added `sableos mode list`
* Added `sableos mode current`
* Added `sableos mode preview <mode>`
* Added support for `work`, `lounge`, `clean`, and `engineering` previews
* Added Ship Mode to the `sableos modules` output

## Changed

* Updated `sableos` to route Ship Mode commands to the repo preview module
* Updated the version to `0.3.0`

## Compatibility Notes

The existing local BEBOP `ship-mode` command remains separate.

SableOS intentionally routes:

```bash
sableos mode ...
```

to the repository preview module instead of the older live desktop script.

This keeps the public SableOS project safe while preserving the existing local desktop workflow.

## Known Limitations

* Ship Mode is preview-only
* No real desktop switching yet
* No mode config files yet
* No backup or rollback system for desktop changes yet
* No uninstall flow yet
