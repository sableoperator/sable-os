# SableOS v0.4.0

Config-driven Ship Mode preview release.

## Summary

SableOS v0.4.0 introduces config-driven Ship Mode previews.

Default Ship Mode descriptions now live in `.mode` config files instead of being hardcoded directly into the Ship Mode script. This makes the default modes feel more like templates and prepares SableOS for future user-defined modes.

Ship Mode remains preview-only and safe.

## What Changed

Ship Mode preview content is now read from config files located at:

```text
modules/ship-mode/config/modes
```

Default mode files include:

```text
work.mode
lounge.mode
clean.mode
engineering.mode
```

The command:

```bash
sableos mode preview work
```

now renders its preview from:

```text
modules/ship-mode/config/modes/work.mode
```

## Current Commands

```bash
sableos mode list
sableos mode current
sableos mode preview work
sableos mode preview lounge
sableos mode preview clean
sableos mode preview engineering
```

## Added

* Added config-driven Ship Mode preview files
* Added default `.mode` files for `work`, `lounge`, `clean`, and `engineering`
* Added Ship Mode config format documentation
* Added support for listing modes from config files
* Added support for rendering previews from config files
* Added helpful error handling for unknown modes
* Added helpful error handling for missing config folders
* Added safer parsing for malformed config fields

## Changed

* Moved default Ship Mode preview language out of the script and into `.mode` files
* Updated Ship Mode documentation to describe default modes as templates
* Updated README and roadmap language for config-driven previews
* Updated version to `0.4.0`

## Safety

This release does not add real desktop switching.

Ship Mode remains preview-only and does not change:

* Wallpapers
* Widgets
* Startup apps
* Cinnamon settings
* Panels
* Windows
* Displays
* Running processes
* User environment

Config actions are printed only. They are not executed.

## Compatibility Notes

The existing local system `ship-mode` command remains separate.

SableOS intentionally routes:

```bash
sableos mode ...
```

to the repository preview module at:

```text
modules/ship-mode/bin/ship-mode
```

This avoids accidentally calling any older system-level `ship-mode` command.

## Known Limitations

* Ship Mode is still preview-only
* No real desktop switching yet
* No user override config path yet
* No backup or rollback system for desktop changes yet
* No uninstall flow yet
* The `.mode` parser is intentionally simple and not a full config language

## Next Goals

* Add a user override config path
* Support custom user-defined modes
