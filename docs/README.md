# SableOS Documentation

This folder contains planning notes, architecture documentation, templates, and safety guides for SableOS.

## Core Documents

- [Project Charter](PROJECT_CHARTER.md)  
  Defines the mission, design principles, and long-term vision of SableOS.

- [Roadmap](ROADMAP.md)  
  Tracks the phased development plan for the project.

- [Module Registry](MODULES.md)  
  Lists official SableOS modules, their status, purpose, and expected locations.

- [System Structure](SYSTEM_STRUCTURE.md)  
  Explains the difference between the real local workstation environment and the clean GitHub repository.

- [Screenshot Guide](SCREENSHOTS.md)  
  Documents how to safely add screenshots without exposing private information.

## Templates

- [Module README Template](templates/MODULE_README_TEMPLATE.md)  
  A reusable template for future SableOS module documentation.

## Documentation Goals

SableOS documentation should make the system easier to:

- understand
- rebuild
- troubleshoot
- extend
- share safely

The documentation should explain not only what each part does, but also why it exists and how it fits into the larger command environment.
